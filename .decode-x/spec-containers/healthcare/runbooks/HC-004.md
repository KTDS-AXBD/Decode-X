# Runbook — HC-004: 예약 상태 전환

## 트리거
- 환자 도착 → checked_in
- 진료 완료 → completed
- 사전 취소 → cancelled
- 미출석 → no_show

## 처리 흐름
1. 현재 status 조회
2. 허용 매트릭스 검증
3. `UPDATE appointments SET status=?, cancelled_at=?` (cancelled 시점만)
4. `{previousStatus, newStatus}` 반환

## 알림
- no_show 마킹 시 환자 SMS/email + 재예약 안내
- completed 시 진료비 결제 트리거 (외부)

## 모니터링
- 일별 상태 전환 카운트 per status
- no_show 비율 (의료기관 운영 입력)
