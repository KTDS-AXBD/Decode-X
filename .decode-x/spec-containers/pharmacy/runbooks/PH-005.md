# Runbook — PH-005: 리콜 대상 약품 일괄 통지

## 트리거
리콜관리SYSTEM (배치) → `POST /api/v1/pharmacy/drugs/mark-recalled`

## 처리 흐름
1. `recalled_at <= recallCutoffDate` AND `status = 'active'` 대상 조회
2. 대상 약품 status='recalled' 일괄 UPDATE
3. markedCount 반환 (0이면 대상 없음, 정상)

## 모니터링
- 리콜 배치 처리 건수 (일별 리콜 약품 수)
- 리콜 약품 잔여 재고 현황
- 리콜 후 조제 시도 차단 건수 (PH-003 E422-DRUG-RECALLED)
