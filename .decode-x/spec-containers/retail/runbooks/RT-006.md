# Runbook — RT-006: 반품 + 환불 Atomic

## 트리거
고객 서비스 반품 요청 → `POST /api/v1/retail/orders/:id/return`

## 처리 흐름
1. orderId로 주문 조회 + status 검증 (delivered or completed)
2. db.transaction: orders.status='returned' + return_log INSERT (전액 환불)

## 모니터링
- 반품률 (returned / completed 비율)
- E409-ORD 발생률 (반품 불가 상태에서 시도)
- 평균 환불 처리 시간
