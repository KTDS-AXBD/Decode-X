# Runbook — CN-002: 보유금 비율 한도 비교

## 트리거
발주처재무팀/대금정산SYSTEM → `POST /api/v1/construction/projects/:id/retentions`

## 처리 흐름
1. 프로젝트 status='in_progress' or 'completed' 확인
2. 계약 규모(contract_value)로 retentionRateLimit 결정
3. `retentionRate > retentionRateLimit` 차단
4. 검증 통과 시 payment_retentions INSERT

## 모니터링
- E422-RETENTION-EXCEEDED 발생률
- 보유금 비율 한도 근접 비율
