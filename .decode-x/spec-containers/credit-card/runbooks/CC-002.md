# Runbook — CC-002: 결제 한도 검증

## 트리거
가맹점 결제 승인 요청 직전 (CC-003 호출 전 사전 검증)

## 처리 흐름
1. `SELECT credit_limit, used_amount, status FROM credit_cards WHERE id=?`
2. `status != 'active'` → `E409-ST`
3. `credit_limit - used_amount < amount` → `E422-LIMIT`
4. `{canPay: true, remainingLimit}` 반환

## 모니터링
- `E422-LIMIT` 발생률 (한도 부족 사용자 비율)
- 평균 remainingLimit / credit_limit 비율
