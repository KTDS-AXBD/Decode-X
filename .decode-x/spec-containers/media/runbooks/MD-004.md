# Runbook — MD-004: 콘텐츠 상태 전환

## 트리거
콘텐츠관리자/콘텐츠SYSTEM → `PATCH /api/v1/media/contents/:id/status`

## 처리 흐름
1. 현재 status 조회
2. 허용 매트릭스 검증 (draft→reviewing→published→archived/expired)
3. contents.status UPDATE + 타임스탬프 기록 (published_at / archived_at)

## 모니터링
- E409-CONTENT 발생률 (잘못된 상태 전환 시도)
- 콘텐츠 심사 사이클 시간 (draft → published 평균)
