# Runbook — CC-003: 결제 승인 atomic

## 트리거
가맹점 결제 승인 요청 (CC-002 통과 후)

## 처리 흐름
1. card 재검증 (race condition 회피 — transaction 내부에서 다시 read)
2. **Atomic transaction** (`db.transaction(() => { ... }); tx();`):
   - `UPDATE credit_cards SET used_amount = used_amount + amount`
   - `INSERT INTO card_transactions (status='approved', approved_at=now)`
3. `{transactionId, remainingLimit, approvedAt}` 반환

## 트러블슈팅
- transaction rollback 시 used_amount 변화 없음 보장
- `E422-LIMIT` 재발생 시 사전 CC-002 검증과 race 의심 (concurrent 결제)

## 모니터링
- 평균 approvePayment 응답 시간 (transaction 비용)
- approval rate vs decline rate
