# Runbook — EN-005: 정전 일괄 통보 배치

## 트리거
통보 SYSTEM 배치 → `POST /api/v1/energy/accounts/{id}/outages/notify`

## 처리 흐름
1. `outage_records WHERE status='pending' AND account_id=?` 조회
2. 각 건 `status='notified'` + `notified_at` UPDATE

## 모니터링
- notifiedCount=0 연속 발생 시 pending backlog 확인
- 통보 지연 (occurred_at ~ notified_at 간격)
