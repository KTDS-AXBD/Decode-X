## 3) Lifecycle 표준 정의

```
                                                ┌──────────┐
                                          ┌────▶│ archived │
신규 INSERT 경로                          │     └──────────┘
─────────────                              │ (수동)
                                           │
queue/handler.ts:300        ─────► draft ──┤      ┌──────────┐
(Stage 5 Skill Packaging)                  ├────▶│ published│  (수동, handleUpdateSkillStatus)
                                           │     └────┬─────┘
skills.ts:654               ─────►reviewed─┤          │
(F362 spec-container 직접)                 │          │
                                           │          ▼
rebundle-orchestrator:154   ─────► bundled │   ┌──────────────┐
(rebundle 결과)                            │   │ superseded   │
                                           ├──▶│ (rebundle 시)│
                                           │   └──────────────┘
                                           │
                                           └─ AI-Ready 평가 대상: bundled + reviewed
```

| Status | 의미 | 진입 경로 | AI-Ready 평가 |
|--------|------|----------|:-------------:|
| `draft` | 신규 생성, 검증 미완 | queue/handler INSERT | ❌ |
| `reviewed` | F362 spec-container → SkillPackage 직접 (Tier-A) | skills.ts:654 INSERT | ✅ |
| `bundled` | Rebundle 결과 SkillPackage 번들 | rebundle-orchestrator:154 | ✅ |
| `published` | 수동 검증 완료 + 배포 승인 | handleUpdateSkillStatus | ✅ (선택, 향후) |
| `superseded` | rebundle로 흡수된 이전 1:1 policy skill | rebundle-orchestrator:178 UPDATE | ❌ (history) |
| `archived` | 영구 폐기 | handleUpdateSkillStatus | ❌ |

> **AI-Ready 평가 대상은 `bundled + reviewed` 2종**. `published`는 향후 수동 검증 워크플로우 연계 시 추가 가능.

## 4) Implementation Steps

### Step 1: D1 Migration 신설 (1h)

**File**: `infra/migrations/db-skill/0013_skills_status_check.sql`

```sql
-- TD-53 해소: skills.status 6-enum CHECK 제약 + 의미 명확화
-- 기존 데이터 호환성 검증 후 적용 (Miraeasset 3065+15 + LPON 859+35 + lpon 8 + org_ktds_axbd 1 = 3983건 전수 6-enum 만족)

-- SQLite는 ALTER TABLE ADD CONSTRAINT 미지원 → 새 테이블 생성 + 데이터 복사 + 교체
CREATE TABLE skills_new (
  skill_id TEXT PRIMARY KEY,
  ontology_id TEXT NOT NULL,
  domain TEXT NOT NULL,
  subdomain TEXT,
  language TEXT NOT NULL DEFAULT 'ko',
  version TEXT NOT NULL,
  r2_key TEXT NOT NULL,
  policy_count INTEGER NOT NULL,
  trust_level TEXT NOT NULL,
  trust_score REAL NOT NULL,
  tags TEXT,
  author TEXT,
  status TEXT NOT NULL DEFAULT 'draft'
    CHECK (status IN ('draft','reviewed','bundled','published','superseded','archived')),
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  content_depth INTEGER NOT NULL DEFAULT 0,
  organization_id TEXT NOT NULL DEFAULT 'unknown',
  spec_container_id TEXT,
  document_ids TEXT
);

INSERT INTO skills_new SELECT * FROM skills;
DROP TABLE skills;
ALTER TABLE skills_new RENAME TO skills;

CREATE INDEX IF NOT EXISTS idx_skills_status ON skills(status);
CREATE INDEX IF NOT EXISTS idx_skills_org_status ON skills(organization_id, status);
```

**검증**: production 적용 전 staging에서 dry-run + row count cross-check (before/after 동일).
