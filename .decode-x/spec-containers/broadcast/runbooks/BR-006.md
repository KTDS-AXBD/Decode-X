# Runbook — BR-006 sponsor 환불 atomic

## 알림 조건
`E404-PREEMPTED-AIRING` 또는 환불 잔액 불일치 (`sponsor_refund_records.status='refunded'`인데 `sponsor_refunds` row 누락) 의심 시.

## 진단
1. `airings.status='preempted'` 조건 충족 확인 (archived 송출은 환불 대상 아님)
2. `refundAmount = sponsorCost × refundRate` 계산 검증 (소수점 2자리 반올림, 광고 makegood 환산 정책 확인)
3. `db.transaction()` rollback 로그 — atomic 실패 시 records/refunds 양쪽 모두 미생성

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: refund rate 정책 (전액/부분/makegood 광고 보전) 별 audit log 추가 + 광고대행사 회신 hook + 방심위 정산 audit
