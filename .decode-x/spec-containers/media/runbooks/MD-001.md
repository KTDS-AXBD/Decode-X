# Runbook — MD-001: 구독 tier 동시 시청 한도 검증

## 트리거
구독자/스트리밍SYSTEM → `POST /api/v1/media/subscriptions/:id/stream`

## 처리 흐름
1. 구독 status='active' 확인
2. `concurrentStreamCount >= MAX_CONCURRENT_STREAMS` 차단 (4 동시 스트림 한도)
3. 스트리밍 세션 활성화

## 모니터링
- E422-STREAM-LIMIT 발생률 (한도 도달 구독자 비율)
- 시간대별 동시 스트림 피크 모니터링
