# BT-006 — 수수료 정산 Atomic 트랜잭션

**함수**: `processCommission()`
**Detector**: AtomicTransaction

## 조건
수수료 정산 요청 시 완료된 예약 확인 후 atomic 처리.

## 기준
`appointments.status = 'completed'` 확인 후 단일 트랜잭션으로:
1. commission_records INSERT (status='calculated')
2. settlements INSERT (status='settled')
3. commission_records UPDATE (status='settled')

## 결과
- 완료 예약: atomic 수수료 정산 (3개 연산 동시 실행)
- 불완료: `E404-COMPLETED-APPOINTMENT` 예외

## 근거
스타일리스트 수수료 분배 + 정산 atomic 처리 — 중간 실패 시 rollback으로 정산 오류 방지.
