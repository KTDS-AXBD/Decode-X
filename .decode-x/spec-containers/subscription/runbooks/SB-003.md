# Runbook — SB-003: 자동 갱신 atomic

## 트리거
정기 batch (분 단위 cron) — `next_charge_at ≤ now AND status='active'` 발견

## 처리 흐름
1. SB-002 한도 검증 (사전)
2. **Atomic transaction**:
   - `INSERT INTO subscription_payments (status='success', cycle_index=N+1)`
   - `UPDATE subscriptions SET cycle_count=N+1, next_charge_at = now + cycle_days`
3. `{paymentId, cycleIndex, nextChargeAt}` 반환

## 트러블슈팅
- atomic rollback 시 cycle_count 변화 없음 + payment 흔적 없음
- next_charge_at 갱신 누락 시 다음 batch에 중복 결제 위험 → atomic 필수

## 모니터링
- 일별 자동 갱신 성공률 (success vs failed)
- 평균 갱신 처리 시간
