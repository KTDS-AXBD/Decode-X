# Runbook — FS-005 stale 주방 티켓 일괄 배치

## 알림 조건
일별/시간별 batch 실행 결과 `staleCount` 급증 (예: 평균 대비 3σ 초과) 시.

## 진단
1. stale 패턴: 특정 매장 / 시간대 / 메뉴 군집 확인
2. preparing 후 ready 전환률 (kitchen_tickets.preparing → ready 비율) 확인
3. 결제 시스템 장애 가능성 확인 (FS-003 atomic 실패와의 연관성)

## 해소
- 임시: stale 티켓 추적 + 고객 환불 + 신규 티켓 재발행 (kitchen_tickets 추가 INSERT)
- 영구: 주방 KDS 시간 알림 + 가맹점 조리시간 SLA 정립 + 메뉴별 표준 조리시간 학습
