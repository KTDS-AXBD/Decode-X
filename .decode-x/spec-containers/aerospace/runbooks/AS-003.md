# Runbook — AS-003 미션 실행 atomic 트랜잭션

## 알림 조건
`E404-MISSION` 빈도 급증 시 또는 mission/payload/payment 정합성 어긋남.

## 진단
1. `missions.status='confirmed'` 보유 row vs payloads/mission_payments 매칭 cross-check
2. 같은 missionId가 multiple payload INSERT 시 transaction 격리 여부 확인
3. better-sqlite3 `db.transaction()` 예외 발생 시 자동 rollback 작동 확인

## 해소
- 즉시: 데이터 정합성 검증 SQL 실행 (missions.payload_id NOT NULL but payloads 부재)
- 영구: missionId unique constraint 강화 또는 payload upsert 시 conflict 정책 명시
