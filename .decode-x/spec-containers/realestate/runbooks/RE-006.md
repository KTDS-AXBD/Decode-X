# Runbook — RE-006: 계약 취소 + 환불 atomic

## 트리거
양 당사자 취소 합의 (`POST /api/v1/realestate/leases/:id/cancel`)

## 처리 흐름
1. lease 조회 (property_id, deposit_krw, start_at, status)
2. status active 검증
3. daysBefore 계산 (start_at - now)
4. refundAmount 산정 (`daysBefore ≥ 14 ? deposit_krw : 0`)
5. **Atomic transaction**:
   - `UPDATE leases SET status='terminated', terminated_at=now`
   - `UPDATE properties SET status='listed'` (재매물화)
6. `{refundAmount, terminatedAt, propertyReleased}` 반환

## 트러블슈팅
- atomic rollback 시 property 'leased' 유지
- 공실 누락 또는 중복 임차 위험 → atomic 필수

## 모니터링
- 일별 취소율
- 14일 이전 vs 미만 환불 비율
