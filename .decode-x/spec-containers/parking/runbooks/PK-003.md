# Runbook — PK-003 입차 atomic

## 알림 조건
`E404-RESERVATION` (confirmed 예약 없음) 빈도 급증 또는 atomic transaction rollback 발생 시.

## 진단
1. `slot_reservations` 상태가 confirmed인 예약 존재 여부
2. atomic transaction 실행 중 어느 단계에서 실패했는지 (parking_sessions INSERT / slot_reservations UPDATE / parking_payments INSERT)
3. 동시성 충돌 (같은 reservation을 두 번 입차 처리하려는 경우) 추적

## 해소
- 임시: 실패 reservation 수동 status 정합화 + parking_payments 누락 시 보정
- 영구: 입차 게이트(physical kiosk) 동시 호출 방지 + idempotency 키 도입
