# Runbook — SM-006 크리에이터 payout 회수 atomic

## 알림 조건
`E404-REMOVED-FEED` 또는 회수 잔액 불일치 (`creator_payout_clawback_records.status='clawed_back'`인데 `creator_payout_clawbacks` row 누락) 의심 시.

## 진단
1. `post_feeds.status='removed'` 조건 충족 확인 (정상 종료된 archived 피드는 회수 대상 아님)
2. `clawbackAmount = payoutCost × clawbackRate` 계산 검증 (소수점 2자리 반올림)
3. `db.transaction()` rollback 로그 — atomic 실패 시 records/clawbacks 양쪽 모두 미생성

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: clawback rate 정책 (TOS 위반 / 저작권 위반 / 가짜 engagement 등 비율 분류) 별 audit log 추가 + 크리에이터 통보 워크플로 audit
