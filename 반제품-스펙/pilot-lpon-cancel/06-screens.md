# 화면 정의 — LPON 온누리상품권

> **도메인**: 온누리상품권 (LPON)
> **범위**: 이용자 / 가맹점 / 관리자 주요 화면
> **수준**: 와이어프레임 + 상태 전이 (후순위 문서)
> **버전**: 1.0
> **작성일**: 2026-03-19

---

## 1. 화면 구성 개요

### 1.1 역할별 화면 분류

| 역할 | 화면 그룹 | 주요 화면 수 |
|------|----------|:----------:|
| USER (이용자) | 상품권, 충전, 결제, 환불, 알림 | 8 |
| MERCHANT (가맹점) | 가맹점 정보, 결제 내역, 정산, 알림 | 5 |
| ADMIN (관리자) | 대시보드, 가맹점 관리, 정산 관리, 환불 관리 | 5 |
| **합계** | | **18** |

## 6. 상태 전이 다이어그램

### 6.1 결제 상태 전이

```mermaid
stateDiagram-v2
    [*] --> PENDING: 결제 요청
    PENDING --> PAID: 잔액 차감 성공
    PENDING --> [*]: 잔액 부족 (실패)
    PAID --> CANCEL_REQUESTED: 사용자 취소 요청
    CANCEL_REQUESTED --> CANCELED: 취소 처리 완료
    CANCELED --> REFUNDED: 환불 완료
    REFUNDED --> [*]
```

### 6.2 환불 상태 전이

```mermaid
stateDiagram-v2
    [*] --> REQUESTED: 환불 신청
    REQUESTED --> APPROVED: 관리자 승인
    REQUESTED --> REJECTED: 관리자 거절
    APPROVED --> COMPLETED: 환불금 복원
    COMPLETED --> [*]
    REJECTED --> [*]
```

### 6.3 상품권 상태 전이

```mermaid
stateDiagram-v2
    [*] --> ISSUED: 상품권 발급
    ISSUED --> ACTIVE: 최초 충전
    ACTIVE --> EXPIRED: 유효기간 만료
    ACTIVE --> SUSPENDED: 관리자 정지
    SUSPENDED --> ACTIVE: 관리자 해제
    EXPIRED --> [*]
```

### 6.4 정산 상태 전이

```mermaid
stateDiagram-v2
    [*] --> PENDING: 정산 주기 도래
    PENDING --> CALCULATED: 금액 계산 완료
    CALCULATED --> CONFIRMED: 관리자 확정
    CALCULATED --> DISPUTED: 가맹점 이의
    CONFIRMED --> PAID: 송금 완료
    DISPUTED --> CALCULATED: 재계산
    PAID --> [*]
```

### 6.5 가맹점 상태 전이

stateDiagram-v2
    [*] --> PENDING: 등록 신청
    PENDING --> ACTIVE: 관리자 승인
    ACTIVE --> SUSPENDED: 관리자 정지
    SUSPENDED --> ACTIVE: 관리자 해제
    ACTIVE --> TERMINATED: 계약 해지
    TERMINATED --> [*]
