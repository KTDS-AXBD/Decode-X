# Runbook — HC-006: 예약 취소 + 환불 atomic

## 트리거
사용자 예약 취소 요청 (`POST /api/v1/healthcare/appointments/:id/cancel`)

## 처리 흐름
1. appointment 조회 (slot_id, fee_krw, scheduled_at, status)
2. status booked/checked_in 검증
3. hoursBefore 계산 (scheduled_at - now)
4. refundAmount 산정 (`hoursBefore ≥ 24 ? fee_krw : 0`)
5. **Atomic transaction**:
   - `UPDATE appointments SET status='cancelled', cancelled_at=now`
   - `UPDATE slots SET status='available'` (slot 해제)
6. `{refundAmount, cancelledAt, slotReleased}` 반환

## 트러블슈팅
- atomic rollback 시 slot 잠금 잔존 위험 → atomic 필수
- 환불은 별도 결제 시스템 호출 (본 함수는 refundAmount 계산만)

## 모니터링
- 일별 취소율 (cancel / 총 예약)
- 24시간 이전 취소 비율 (환불 정책 효과 검증)
- slot 재배정률 (cancel 후 재예약 패턴)
