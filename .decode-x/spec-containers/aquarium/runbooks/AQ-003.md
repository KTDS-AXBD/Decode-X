# Runbook — AQ-003 관람 입장 atomic

## 알림 조건
`E404-SCHEDULE` 또는 atomic 부분 실패 (admits INSERT 성공 + admit_payments INSERT 실패) 의심 시.

## 진단
1. `tour_schedules.status` 확인 — `reserved` 상태에서만 입장 진입 가능
2. `db.transaction()` rollback 로그 확인
3. `admits.schedule_id` ↔ `admit_payments.schedule_id` 정합성 cross-check + 게이트 reader/QR 스캐너 모니터링

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: cross-table FK 강제 + 입장 reader fallback (백업 단말) + 모바일 admit dual-channel 설계
