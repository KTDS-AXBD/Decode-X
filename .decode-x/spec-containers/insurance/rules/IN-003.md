# IN-003: 청구 승인 atomic transaction

**Rule**: pending 상태 검증 후 atomic하게 status='approved' + reserved_amount UPDATE

**구현 함수**: `approveClaim`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();`)

## 트리거 조건
- **When**: 심사관 승인 결정 또는 자동 심사 통과
- **If**: claim status='pending' + policy active + 한도 충분
- **Then**: atomic transaction
  1. `UPDATE claims SET status='approved', approved_at=now`
  2. `UPDATE policies SET reserved_amount = reserved_amount + claim_amount`
- **Else**: `E404-CL` / `E409-CL` / `E409-PO` / `E422-LIMIT`

## 무결성 보장
- claim 승인 + policy reserved_amount 갱신 atomic — 한도 누적 누락 방지
- atomic rollback 시 두 변경 모두 무효화

## 신뢰도
- Detector: 85% (AtomicTransaction — better-sqlite3 표준)
