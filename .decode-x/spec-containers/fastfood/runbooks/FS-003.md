# Runbook — FS-003 결제 atomic

## 알림 조건
`E404-ORDER` (confirmed 주문 없음) 빈도 급증 또는 atomic transaction rollback 발생 시.

## 진단
1. `orders` 상태가 confirmed인 주문 존재 여부
2. atomic transaction 단계 실패 (kitchen_tickets INSERT / orders UPDATE / order_payments INSERT)
3. 동시성 충돌 (같은 order 두 번 결제 처리) 추적

## 해소
- 임시: 실패 order 수동 status 정합화 + order_payments 누락 시 보정
- 영구: 결제 인증(NFC/카드) 동시 호출 방지 + idempotency 키 도입
