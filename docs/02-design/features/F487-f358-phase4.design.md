## §3. Risk Analysis

### R1 — LPON Java 소스 미보유 (Sprint 258 발견, Phase 3b 분할 사유)

**Status**: Phase 4 진입 시점에도 동일 제약.

**Mitigation 3-Way**:
- **Option A (R2 check)**: `wrangler r2 object list --prefix=documents/lpon/ --env production` — 기 업로드된 Java 소스 확인
- **Option B (PoC samples)**: Sprint 257 Tree-sitter PoC 5 샘플 spec-container 재사용 (8 containers available)
- **Option C (Partial + TD-XX)**: DIVERGENCE 재실측 부분 종결 + 신규 TD 등록

**판정 (Sprint 321 실측)**:
- 8 LPON spec-containers available locally (lpon-{refund,charge,payment,gift,settlement,budget,purchase,cancel})
- DIVERGENCE 재실측: detect-bl TypeScript spec-container 기준으로 실행 가능 → **Option B 적용**
- Production 재추출: LPON 0 approved policies in D1 (Sprint 267 시점) — production 실행은 Master 위임

### R2 — F356-A 평가 입력 한계 (AIF-PLAN-056 발견)

**Status**: Tree-sitter 통합 후 spec-container에 반영 여부 불확실.

**Mitigation**: baseline avg 0.506 (세션 264) 유지로 무영향 종결. Master 실행 시 `--force=true` 재평가.

### R3 — autopilot Production Smoke 14회차 변종

**Mitigation 5-Layer (Plan §(b)~(e))**:
- (b) Multi-input 9 probe: curl 스크립트 준비 → Master 실행
- (c) wrangler tail 30s: Master 실행 후 증거 첨부
- (d) reports 4종 실파일: Sprint 321 보고서 파일 생성 (detect-bl 결과 기반)
- (e) Master ps+curl 독립 검증: post-merge Master 위임
