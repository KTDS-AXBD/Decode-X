# Runbook — BK-001: 일일 출금 한도 검증

## 트리거
고객/출금SYSTEM → `POST /api/v1/banking/accounts/:id/withdraw`

## 처리 흐름
1. 계좌 status='active' 확인
2. `amount >= MAX_WITHDRAWAL_AMOUNT` 차단 (1천만원 한도)
3. `accounts.balance >= amount` 확인
4. `transactions` INSERT + `accounts.balance` 차감

## 모니터링
- E422-WITHDRAWAL-LIMIT 발생률 (한도 도달 고객 비율)
- E422-INSUFFICIENT-FUNDS 발생률 (잔액 부족 비율)
