# Runbook — AS-006 미션 abort 환불 atomic

## 알림 조건
`E404-LAUNCHING-PAYLOAD` 빈도 급증 시 또는 abort_refund_records/abort_refunds 정합성 어긋남.

## 진단
1. `payloads.status='launching'` 보유 row 분포 확인 (abort 대상)
2. abort_refund_records.status = 'calculated' 이지만 abort_refunds 미INSERT 시 transaction 미완료
3. refund_amount 계산 정확성 확인 (`Math.round(missionCost * refundRate * 100) / 100`)
4. better-sqlite3 `db.transaction()` 예외 발생 시 자동 rollback 작동 확인

## 해소
- 즉시: 정합성 검증 SQL 실행 (abort_refund_records 'calculated' but abort_refunds 부재 row reconcile)
- 영구: abort_refund_records unique constraint 강화 또는 refund upsert 시 conflict 정책 명시
