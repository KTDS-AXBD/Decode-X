# Runbook — SB-006: 취소 + 환불 atomic

## 트리거
사용자 환불 요청 (`POST /api/v1/subscriptions/:id/cancel-with-refund`)

## 처리 흐름
1. subscription 조회 (price_krw, started_at, status)
2. status NOT IN {cancelled, expired} 검증
3. elapsedDays 계산 (started_at vs now)
4. refundAmount 산정 (`elapsedDays ≤ 14 ? price_krw : 0`)
5. **Atomic transaction**:
   - `UPDATE subscriptions SET status='cancelled', cancelled_at=now`
   - (조건부) `INSERT INTO subscription_payments (status='refunded', amount=-refundAmount)`
6. `{refundPaymentId, refundAmount, cancelledAt}` 반환

## 트러블슈팅
- atomic rollback 시 status 변화 없음 + refund payment 흔적 없음
- refundAmount 음수 표기 (ledger 일관성 — 차감 표시)

## 모니터링
- 일별 환불률 (cancelWithRefund / 총 cancelled)
- 평균 elapsedDays + 14일 이내 환불 비율
- 누적 환불 비용 (회계 입력)
