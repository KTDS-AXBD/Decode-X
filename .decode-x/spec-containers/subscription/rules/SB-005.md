# SB-005: 만료 자동 처리

**Rule**: active 상태 + expires_at 도달 시 status='expired' 자동 마킹

**구현 함수**: `markExpiredSubscriptions`
**Detector**: StatusTransition (`status='expired'` 일괄 UPDATE 패턴, CC-005/DV-005 동일 형태)

## 트리거 조건
- **When**: 정기 batch (일/시간 단위, SYSTEM cron)
- **If**: `status='active'` AND `expires_at IS NOT NULL` AND `expires_at < now`
- **Then**: `UPDATE subscriptions SET status='expired'`, 마킹된 subscriptionIds 반환
- **Else**: 만료 안 됐거나 expires_at NULL 시 제외

## 모니터링
- 일별 expired 마킹 카운트 (대시보드)
- 만료 후 재가입률 (CRM 입력)

## 신뢰도
- Detector: 75% (StatusTransition — batch UPDATE, CC-005/DV-005 입증된 동일 형태)
