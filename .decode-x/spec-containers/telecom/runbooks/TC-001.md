# Runbook — TC-001: 가입 회선 활성화 동시 한도 검증

## 트리거
고객/가입SYSTEM → `POST /api/v1/telecom/subscriptions/activate`

## 처리 흐름
1. 고객(`customer_id`) 기준 활성(status='active') 회선 건수 조회
2. `activeLines >= MAX_ACTIVE_LINES` 차단 (5회선 한도)
3. `subscriptions` INSERT + status='active'

## 모니터링
- E422-LINE-LIMIT 발생률 (한도 도달 고객 비율)
