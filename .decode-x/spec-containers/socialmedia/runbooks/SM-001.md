# Runbook — SM-001 계정 동시 active post 한도

## 알림 조건
`E422-ACCOUNT-CAPACITY-EXCEEDED` 빈도 급증 시.

## 진단
1. `accounts.active_published_posts` vs `total_capacity` 비교 조회
2. 특정 계정/카테고리/이벤트(라이브 스트리밍, 챌린지, 트렌딩) 동시 post 집중 여부 확인
3. `MAX_CONCURRENT_ACTIVE_POSTS_PER_ACCOUNT` 기본값(10000) 적절성 검토 (개인 vs 브랜드 계정)

## 해소
- 임시: 계정에 archived 후보 post 정리 권유 또는 total_capacity 직접 조정 (브랜드 계정 상향)
- 영구: 콘텐츠 정원 정책 명문화 + 자동 archive 정책 (장기 미참여 post는 archive 자동 전환)
