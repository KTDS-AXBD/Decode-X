# HC-003: 예약 잡기 atomic transaction

**Rule**: slot 가용 검증 후 atomic하게 appointment INSERT + slot reserved UPDATE

**구현 함수**: `bookAppointment`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();`)

## 트리거 조건
- **When**: 사용자 예약 요청
- **If**: `slot.status='available'`
- **Then**: atomic transaction
  1. `INSERT INTO appointments (status='booked')`
  2. `UPDATE slots SET status='reserved'`
- **Else**: `E404-SL` / `E409-SL`

## 무결성 보장
- 두 변경 atomic — slot reserved 누락 시 중복 예약 위험
- atomic rollback 시 두 변경 모두 무효화

## 신뢰도
- Detector: 85% (AtomicTransaction)
