# DV-004: 배송 상태 전환 (SHIPPED → DELIVERED/CANCELLED/RETURNED, PENDING → CANCELLED)

**Rule**: 허용된 상태 전환만 인정. 불허 전환 시 E409-TR

**구현 함수**: `transitionDeliveryStatus`
**Detector**: StatusTransition (`status` SELECT-then-UPDATE 패턴)

## 트리거 조건
- **When**: 배송 시스템 또는 사용자 상태 변경 요청
- **If**: 허용 매트릭스 충족
  - `PENDING → CANCELLED` (사용자 사전 취소)
  - `SHIPPED → DELIVERED` (정상 완료)
  - `SHIPPED → CANCELLED` (배송 중 취소)
  - `SHIPPED → RETURNED` (반품)
- **Then**: `UPDATE deliveries SET status=?, <field>=?` (delivered_at/cancelled_at)
- **Else**: `E409-TR` (불허 전환, 예: DELIVERED → SHIPPED)

## 상태 머신
```
PENDING → CANCELLED / SHIPPED
SHIPPED → DELIVERED / CANCELLED / RETURNED / DELAYED (DV-005 batch)
DELIVERED → RETURNED (DV-006)
```

## 신뢰도
- Detector: 75% (StatusTransition — SELECT status + UPDATE status)
