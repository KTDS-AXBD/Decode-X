# Runbook — BR-003 broadcast 송출 atomic

## 알림 조건
`E404-SCHEDULE` 또는 atomic 부분 실패 (airings INSERT 성공 + sponsor_payments INSERT 실패) 의심 시.

## 진단
1. `broadcast_schedules.status` 확인 — `scheduled` 상태에서만 송출 진입 가능
2. `db.transaction()` rollback 로그 확인 (better-sqlite3 transaction은 동기 atomic)
3. `airings.schedule_id` ↔ `sponsor_payments.schedule_id` 정합성 cross-check + 실시간 송출 인프라 모니터링

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: cross-table FK 강제 + audit log 추가 (schedule_id 단일성 보장) + 송출 인프라 fallback 설계 (백업 송출소)
