# SB-003: 자동 갱신 atomic transaction

**Rule**: active 상태 검증 후 atomic하게 payment INSERT + cycle_count++ + next_charge_at UPDATE

**구현 함수**: `autoRenew`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();`)

## 트리거 조건
- **When**: 정기 batch가 `next_charge_at ≤ now` 발견 시 SB-002 통과 후
- **If**: `sub.status='active'`
- **Then**: atomic transaction
  1. `INSERT INTO subscription_payments (status='success', cycle_index=N+1)`
  2. `UPDATE subscriptions SET cycle_count=N+1, next_charge_at = now + cycle_days`
- **Else**: `E404-SB` 또는 `E409-ST`

## 무결성 보장
- payment INSERT + cycle 증가 atomic — 결제 성공 후 cycle 미반영 방지
- next_charge_at 갱신 누락 시 중복 결제 위험 → atomic 필수

## 신뢰도
- Detector: 85% (AtomicTransaction — better-sqlite3 표준)
