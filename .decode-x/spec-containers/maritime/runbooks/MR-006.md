# Runbook — MR-006: 손상 신고 atomic

## 트리거
손해사정인/손상관리SYSTEM → `POST /api/v1/maritime/shipments/:id/damage-claim`

## 처리 흐름
1. 선적 status='delivered' 또는 'arrived' 확인
2. atomic 트랜잭션:
   a. damage_claims INSERT (status='compensated')
   b. compensation_records INSERT
   c. shipments.damage_claim_filed=1 갱신

## 모니터링
- 손상 신고 건수 및 금액 분포
- E409-SHIPMENT 발생률 (비정상 상태에서 신고 시도)
- 보상 처리 시간 (submitted_at → processed_at)
