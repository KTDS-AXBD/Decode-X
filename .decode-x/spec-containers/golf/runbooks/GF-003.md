# Runbook — GF-003 티오프 atomic

## 알림 조건
`E404-SCHEDULE` 또는 atomic 부분 실패 (rounds INSERT 성공 + round_payments INSERT 실패) 의심 시.

## 진단
1. `tee_schedules.status` 확인 — `reserved` 상태에서만 티오프 진입 가능
2. `db.transaction()` rollback 로그 확인
3. `rounds.schedule_id` ↔ `round_payments.schedule_id` 정합성 cross-check + 1번 홀 스타터 인프라 모니터링

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: cross-table FK 강제 + 결제 PG fallback + 스타터 dual-channel 운영 설계 (전산+무전)
