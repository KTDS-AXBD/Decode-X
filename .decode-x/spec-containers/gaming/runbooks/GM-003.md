# Runbook — GM-003 게임 세션 atomic 실패

## 알림 조건
`E404-LAUNCH` 빈도 급증 또는 atomic 트랜잭션 rollback 감지.

## 진단
1. `game_launches.status = 'published'` 상태 row 존재 여부 확인
2. `game_sessions`, `store_payments` 동기화 정합 (각 trio 1:1:1)
3. db.transaction() race condition 또는 lock contention 추적 (라이브 서비스 동시 접속 폭주)

## 해소
- 임시: 실패 launch 수동 status 보정 + 재시도
- 영구: published → live 전환 idempotent 보장 + DLQ 도입 검토 + 동시성 제한 도입
