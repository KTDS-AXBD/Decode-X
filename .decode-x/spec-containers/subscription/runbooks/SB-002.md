# Runbook — SB-002: 자동결제 한도 검증

## 트리거
SB-003 자동 갱신 직전 (cron이 next_charge_at 도달 발견)

## 처리 흐름
1. `SELECT price_krw, auto_charge_limit, status FROM subscriptions WHERE id=?`
2. `status != 'active'` → `E409-ST`
3. `auto_charge_limit < price_krw` → `E422-LIMIT`
4. `{canCharge: true, remainingHeadroom}` 반환

## 모니터링
- E422-LIMIT 발생률 (사용자 한도 부족 비율)
- 평균 remainingHeadroom (구매력 대시보드)
