# SB-006: 취소 + 환불 atomic

**Rule**: 비종료 상태 검증 후 atomic하게 status='cancelled' + 14일 이내 환불 INSERT

**구현 함수**: `cancelWithRefund`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();`)

## 트리거 조건
- **When**: 사용자 환불 요청
- **If**: `sub.status NOT IN {cancelled, expired}`
- **Then**: atomic transaction
  1. `UPDATE subscriptions SET status='cancelled', cancelled_at=now`
  2. (조건부) `INSERT INTO subscription_payments (status='refunded', amount=-price_krw)` if `elapsedDays ≤ MAX_REFUND_DAYS`
- **Else**: `E404-SB` 또는 `E409-SB` (이미 종료)

## 환불 정책
- 14일 이내 → 100% 환불 (`amount = -price_krw`)
- 14일 초과 → 0원 (refund payment 미생성)

## 무결성 보장
- 취소 + 환불 atomic — 취소만 성공하고 환불 누락되는 사고 방지
- refund_amount 음수 표기로 차감 ledger 일관성

## 신뢰도
- Detector: 85% (AtomicTransaction — better-sqlite3 표준)
