# Runbook — HC-005: 약제 만료 자동 처리

## 트리거
정기 batch (시간 단위, SYSTEM cron)

## 처리 흐름
1. 후보 조회: `status='active'` AND `expires_at < now`
2. 각 prescriptionId에 대해:
   - `UPDATE prescriptions SET status='expired'`
3. `{markedCount, expiredPrescriptionIds}` 반환

## 알림
- expired 마킹 시 환자 SMS/email + 재처방 안내
- markedCount > 일평균 × 2 시 dashboard alert

## 모니터링
- 일별 expired 신규 마킹 카운트
- drug_code별 만료율 (안 쓴 약 패턴 분석)
