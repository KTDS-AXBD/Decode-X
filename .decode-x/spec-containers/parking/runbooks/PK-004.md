# Runbook — PK-004 예약 상태 전환 매트릭스

## 알림 조건
`E409-RESERVATION` (허용 매트릭스 위반 transition 시도) 빈도 급증 시.

## 진단
1. 시도된 (previousStatus → newStatus) 쌍 분포 조회
2. 비정상 전환 시도가 UI 클라이언트 버그인지 외부 API 호출인지 구분
3. 동시성으로 인한 stale read 가능성 확인

## 해소
- 임시: 시도 차단 + 사용자 안내 메시지 ("이미 처리된 예약입니다")
- 영구: 클라이언트 상태 머신을 백엔드 매트릭스와 1:1 동기화 + optimistic locking
