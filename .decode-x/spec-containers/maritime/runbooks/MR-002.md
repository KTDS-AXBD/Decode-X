# Runbook — MR-002: 운임 한도 비교

## 트리거
운임팀/운임계산SYSTEM → `POST /api/v1/maritime/shipments/:id/freight-rate`

## 처리 흐름
1. 선적 정보 조회 (origin_port, dest_port)
2. 항로 유형 결정 (intra_asia / asia_europe / transpacific / transatlantic)
3. `quotedRate > freightRateLimit` (항로별 한도) 차단
4. 검증 통과 시 freight_rates INSERT

## 모니터링
- E422-FREIGHT-RATE-EXCEEDED 발생률 (운임 한도 초과 비율)
- 항로별 운임 분포 통계
- freightRateLimit 근접 비율 (>90%)
