# Runbook — DV-003: 배송 시작 atomic

## 트리거
배송 시스템 출고 처리 (DV-002 통과 후)

## 처리 흐름
1. status PENDING 검증
2. **Atomic transaction**:
   - `UPDATE deliveries SET status='SHIPPED', shipped_at=now`
   - `INSERT INTO delivery_events (event_type='SHIPPED')`
3. `{deliveryId, status: 'SHIPPED', shippedAt}` 반환

## 트러블슈팅
- transaction rollback 시 status PENDING 유지 보장
- delivery_events SHIPPED 누락 시 추적 불가

## 모니터링
- 일별 출고 카운트 (지역/시간별)
