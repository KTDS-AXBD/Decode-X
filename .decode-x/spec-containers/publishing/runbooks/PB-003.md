# Runbook — PB-003 인쇄 batch atomic 실패

## 알림 조건
`E404-REGISTRATION` 빈도 급증 또는 atomic 트랜잭션 rollback 감지.

## 진단
1. `volume_registrations.status = 'edited'` 상태 row 존재 여부 확인
2. `print_batches`, `royalty_payments` 동기화 정합 (각 trio 1:1:1)
3. db.transaction() race condition 또는 lock contention 추적

## 해소
- 임시: 실패 registration 수동 status 보정 + 재시도
- 영구: edited → printed 전환 idempotent 보장 + DLQ 도입 검토
