# Runbook — TC-003: 플랜 업그레이드 atomic 트랜잭션

## 트리거
고객/플랜SYSTEM → `POST /api/v1/telecom/subscriptions/:id/upgrade`

## 처리 흐름
1. 회선 조회 + status='active' 검증
2. 신규 요금제 존재 확인
3. atomic tx:
   - `subscriptions.plan_id` UPDATE
   - `billing_cycles` INSERT (upgrade 당월 청구)
   - `subscriptions.activated_at` 갱신

## 모니터링
- 플랜 업그레이드 성공/실패율
- E409-SUBSCRIPTION (비활성 회선 변경 시도)
