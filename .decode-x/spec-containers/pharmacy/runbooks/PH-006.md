# Runbook — PH-006: 약물 상호작용 atomic

## 트리거
약사/약물안전SYSTEM → `POST /api/v1/pharmacy/prescriptions/:id/check-interaction`

## 처리 흐름
1. 처방 status != 'completed' && != 'expired' 확인
2. severe 상호작용 조회 (interactions 테이블)
3. atomic:
   - severe 발견 시: interaction_logs INSERT (blocked=1) + prescriptions status 재조정
   - 없으면: interaction_logs INSERT (blocked=0)
4. 대체약 정보 반환

## 모니터링
- severe 상호작용 발생률 (약품 조합별)
- 대체약 추천 수락률
- 상호작용 검증 후 처방 차단 비율
