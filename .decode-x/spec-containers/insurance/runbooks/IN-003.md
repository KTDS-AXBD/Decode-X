# Runbook — IN-003: 청구 승인 atomic

## 트리거
심사관 승인 결정 (`POST /api/v1/insurance/claims/:id/approve`)

## 처리 흐름
1. claim 조회 (policy_id, claim_amount, status)
2. status pending 검증
3. policy 재조회 + 한도 재검증 (race 회피)
4. **Atomic transaction**:
   - `UPDATE claims SET status='approved', approved_at=now`
   - `UPDATE policies SET reserved_amount = reserved_amount + claim_amount`
5. `{claimId, approvedAmount, approvedAt}` 반환

## 트러블슈팅
- atomic rollback 시 두 변경 모두 무효화
- 한도 누적 누락 시 후속 청구에서 한도 초과 missed → atomic 필수

## 모니터링
- 일별 승인 건수 + 승인률 (approved / total filed)
- 평균 approvedAmount + 누적 reserved_amount per policy_type
