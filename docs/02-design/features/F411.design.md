# F411 Zip 내부 spec coverage 가시화 — Design

> **Summary**: zip-extractor → source-aggregator → handleGetTriage 3 레이어를 통과하는 chunk metadata 흐름에 (a) 5개 stats 필드 추가, (b) `SourceProjectSummary` element_type 처리 신규, (c) Triage API 응답에 `chunkSummary`/`isLibOnly`/`partialExtraction` 노출 3개 변경을 가한다. UI는 zip 문서 row에 expand 매트릭스 + lib-only/partial badge를 추가하여 3목표(가시화/lib-only/부분추출)를 한 번에 충족.
>
> **Plan**: `docs/01-plan/features/F411.plan.md` (AIF-PLAN-042)
> **Sprint**: 240

## 5. Rollout / Rollback

### 5.1 Rollout 순서
1. **Backend 배포** (zip-extractor + aggregator + analysis): production 배포 즉시 신규 zip 업로드부터 5 stats 필드 채워짐. 기존 27 zip은 chunkSummary undefined → UI fallback "—" 정상 동작.
2. **Frontend 배포** (Pages): backend 신규 응답 필드 안전하게 처리(optional 모두). 신규 UI는 chunkSummary 있을 때만 활성.
3. **Backfill** (선택, 후속 TD-52): 기존 27 zip backfill 스크립트는 별도 Sprint. 본 F411 범위 외.

### 5.2 Rollback
- Backend: 이전 commit으로 git revert. chunk JSON에 신규 필드 추가는 backward-compatible(JSON.parse 무영향).
- Frontend: 이전 commit revert. 신규 UI 전부 제거되고 기존 row 그대로.
- D1 schema 변경 0건이므로 migration rollback 불필요.

### 5.3 Feature Flag
필요 없음 — chunkSummary optional 처리로 자연스러운 점진 전환.
