# CC-003: 결제 승인 atomic transaction

**Rule**: 한도 검증 통과 시 atomic하게 used_amount 증가 + card_transactions INSERT

**구현 함수**: `approvePayment`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();` 패턴)

## 트리거 조건
- **When**: 결제 승인 요청
- **If**: `card.status='active'` AND `remainingLimit ≥ amount`
- **Then**: atomic transaction
  1. `UPDATE credit_cards SET used_amount = used_amount + ?`
  2. `INSERT INTO card_transactions (status='approved', approved_at=now)`
- **Else**: `E404`/`E409-ST`/`E422-LIMIT` 중 첫 위반

## 무결성 보장
- 두 UPDATE/INSERT 모두 transaction 내부 → 실패 시 자동 rollback
- used_amount 음수 발생 불가 (결제는 always 양수 amount)

## 신뢰도
- Detector: 85% (AtomicTransaction — better-sqlite3 표준 `db.transaction()`)
