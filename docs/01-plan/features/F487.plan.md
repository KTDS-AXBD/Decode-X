## Risks & Mitigations

### R1 — LPON 실 Java 소스 미보유 (Sprint 258 발견)
- **Risk**: Phase 3a 분할 사유였음. Tree-sitter parser는 Java 소스 입력 필요.
- **Mitigation**:
  - Option A — R2 `documents/lpon/` prefix 확인 후 기 업로드된 소스 활용
  - Option B — Sprint 257 `wrangler dev` PoC 5 샘플 재사용 (LPON 5 컨테이너만 cover, 35 미달)
  - Option C — autopilot이 LPON 소스 미발견 시 부분 결과 + TD-XX 신규 등록 + 부분 종결

### R2 — F356-A 평가 입력 메커니즘 한계
- **Risk**: AIF-PLAN-056 발견대로 F356-A 평가 입력은 R2 `.skill.json` policies로 Tree-sitter 결과 직접 반영 불가 가능성
- **Mitigation**: Step 4 시 `force=true` + Tree-sitter 산출물이 policies에 반영된 후 재평가. 미반영 시 baseline 0.506 그대로 유지로 무영향 종결.

### R3 — autopilot Production Smoke 14회차 변종 재현
- **Risk**: autopilot이 self "Match 100% 완료" 보고 + reports 실파일 미생성 + 5xx 미인식
- **Mitigation**: §Objective (b)~(e) 5중 가드. 특히 Master ps+curl 독립 검증이 최후 진실 게이트.

### R4 — production deploy 5xx 발생 시 즉시 revert 결정
- **Risk**: S341 16회차 변종 — production smoke 1건이라도 5xx면 4h 41m downtime 사례
- **Mitigation**: Multi-input 9 probe 중 1건이라도 5xx 응답 시 즉시 사용자 인터뷰 → revert vs hotfix forward 결정. master HEAD revert PR 5분 내 자동 생성 + auto-merge.
