# Runbook — AD-006 환불 (chargeback) atomic 실패

## 알림 조건
`E404-CANCELED-BATCH` 빈도 급증 또는 chargeback refund 트랜잭션 rollback 감지.

## 진단
1. `impression_batches.status = 'canceled'` 상태 row 존재 여부 확인
2. `chargeback_refund_records`, `chargeback_refunds` 동기화 정합 (각 1:1)
3. refund_amount 산정 정확도 (media_cost × refund_rate × 100 / 100 rounding)

## 해소
- 임시: 실패 환불 수동 status 보정 + 재시도
- 영구: canceled → refunded atomic 보장 + 환불 정책 변경 시 회귀 테스트
