# Runbook — IN-005: 만료 자동 처리

## 트리거
정기 batch (시간 단위, SYSTEM cron)

## 처리 흐름
1. 후보 조회: `status='active'` AND `expires_at IS NOT NULL` AND `expires_at < now`
2. 각 policyId에 대해:
   - `UPDATE policies SET status='expired'`
3. `{markedCount, expiredPolicyIds}` 반환

## 알림
- expired 마킹 시 사용자 SMS/email + 갱신 유도 캠페인 트리거
- markedCount > 일평균 × 2 시 dashboard alert (이상 신호)

## 모니터링
- 일별 expired 신규 마킹 카운트
- 만료 후 갱신률 (CRM 입력)
