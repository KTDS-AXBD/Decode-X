# IN-006: 청구 거절 + 부분 환불 atomic

**Rule**: pending claim 거절 시 atomic하게 status='rejected' + 30일 이내 시 premium 1회분 환불

**구현 함수**: `rejectClaimWithRefund`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();`)

## 트리거 조건
- **When**: 심사관 거절 결정
- **If**: claim status='pending'
- **Then**: atomic transaction
  1. `UPDATE claims SET status='rejected', rejected_at=now`
  2. (조건부) `INSERT INTO claim_payments (payment_type='refund', amount=premium_krw)` if `elapsedDays ≤ MAX_REFUND_DAYS`
- **Else**: `E404-CL` / `E409-CL`

## 환불 정책
- 30일 이내 청구 거절 → premium 1회분 환불 (미감정 보전)
- 30일 초과 → 0원 환불

## 무결성 보장
- claim 거절 + 환불 atomic — 거절 후 환불 누락 방지
- refundPaymentId NULL 인 경우 환불 INSERT 자체 skip (atomic 자체는 status update만 commit)

## 신뢰도
- Detector: 85% (AtomicTransaction — better-sqlite3 표준)
