# Runbook — AM-001 테마파크 동시 active ticket 한도

## 알림 조건
`E422-PARK-CAPACITY-EXCEEDED` 빈도 급증 시.

## 진단
1. `parks.active_tickets` vs `total_capacity` 비교 조회
2. 특정 파크/성수기(주말/공휴일/방학) 동시 입장 집중 여부 확인
3. `MAX_CONCURRENT_ACTIVE_TICKETS_PER_PARK` 기본값(5000) 적절성 검토 (안전 수용 인원 기준)

## 해소
- 임시: 파크에 completed 후보 ticket 정리 권유 또는 total_capacity 직접 조정
- 영구: 시간대 분산 입장권/예약제 도입 + 추가 어트랙션 증설 + 안전 수용 인원 재산정
