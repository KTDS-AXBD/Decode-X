# Runbook — V-006: 양도

## 트리거
- `POST /api/v1/vouchers/{id}/transfer` 호출

## 처리 흐름
1. voucher row 조회
2. transferred_count >= 1 → E423-LOCKED throw
3. status !== 'active' → E409-STATUS throw
4. UPDATE owner_id, transferred_count+=1, status='transferred'
5. (옵션) 신/구 owner 알림 발송

## 모니터링
- 양도 시도 거부율 (LOCKED vs STATUS)
- 양도 후 사용 비율 (양도 후 미사용 잔액 추적)
