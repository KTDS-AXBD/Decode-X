# Runbook — P-007: 원리금 지급 원자성

**Rule**: 원금 차감 + 이자 지급 + 이력 기록을 단일 트랜잭션으로 처리

## 트리거
- 만기 수령, 중도인출 확정, 해지 환급 등 P-007 호출 시

## 전제조건
- `principal`: 지급할 원금 금액 (양수)
- `interest`: 지급할 이자 금액 (0 이상)
- `disbursementType`: 지급 유형 (`MATURITY` | `EARLY_WITHDRAWAL` | `TERMINATION`)

## 처리 흐름
1. `principal <= 0` → `E422-INVALID_PRINCIPAL` 반환 (트랜잭션 없음)
2. `db.transaction()` 시작:
   a. `pension_ledger.balance -= principal` (원금 차감)
   b. `pension_payouts` INSERT (principal + interest + type)
   c. `PrincipalInterestDisbursed` 이벤트 준비
3. 트랜잭션 커밋 실패 → 전체 롤백 (ES-PENSION-003)

## 에스컬레이션
- ES-PENSION-003: 트랜잭션 실패 반복(3회 이상) 시 수기처리 에스컬레이션 + 감사 로그 보존
