# Runbook — RE-005: 계약 만료 자동 처리

## 트리거
정기 batch (시간 단위, SYSTEM cron)

## 처리 흐름
1. 후보 조회: `status='active'` AND `end_at < now`
2. 각 leaseId에 대해:
   - `UPDATE leases SET status='expiring'`
3. `{markedCount, expiringLeaseIds}` 반환

## 알림
- expiring 마킹 시 양 당사자 통보 + 갱신 안내 캠페인
- markedCount > 일평균 × 2 시 dashboard alert

## 모니터링
- 일별 expiring 신규 마킹 카운트
- 갱신/해지 분포 (CRM 입력)
