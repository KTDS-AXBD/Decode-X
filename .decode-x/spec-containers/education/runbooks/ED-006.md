# Runbook — ED-006: 수강 취소 + 환불 atomic

## 트리거
학생 수강 취소 (`POST /api/v1/education/enrollments/:id/withdraw`)

## 처리 흐름
1. enrollment 조회 (course_id, fee_krw, status)
2. status enrolled 검증
3. daysBefore 계산 (term_start - now)
4. refundAmount 산정 (`daysBefore ≥ 7 ? fee_krw : 0`)
5. **Atomic transaction**:
   - `UPDATE enrollments SET status='withdrawn', withdrawn_at=now`
   - `UPDATE courses SET enrolled_count = enrolled_count - 1`
6. `{refundAmount, withdrawnAt, courseSpotReleased}` 반환

## 트러블슈팅
- atomic rollback 시 course capacity 정합 보장
- refund는 별도 결제 시스템 호출 (본 함수는 amount 계산만)

## 모니터링
- 학기별 withdraw 비율
- 7일 이전 vs 미만 환불 비율
