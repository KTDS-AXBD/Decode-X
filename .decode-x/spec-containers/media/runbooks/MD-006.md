# Runbook — MD-006: 콘텐츠 테이크다운 atomic

## 트리거
콘텐츠모더레이션SYSTEM → `POST /api/v1/media/contents/:id/takedown`

## 처리 흐름
1. content.status 확인 (archived/expired면 E409)
2. report 존재 확인
3. atomic:
   a. contents.status='archived' + archived_at
   b. reports.status='resolved' + resolved_at
   c. licenses WHERE status='active' → status='revoked'
   d. refunds INSERT (무효화 라이선스 건수만큼)
4. refundCount 반환

## 모니터링
- 테이크다운 건수 추이 (저작권 침해 / 불법 콘텐츠 신고)
- 환불 처리 건수 및 환불 금액
- 테이크다운 후 사용자 이의 신청 비율
