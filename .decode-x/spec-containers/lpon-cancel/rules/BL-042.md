# BL-042: 결제 망 취소 처리

**Rule**: 외부 결제망에서 취소 이벤트 수신 시 멱등성 키 기반 중복 처리 방지 + atomic transaction으로 payments/cancel_transactions/vouchers 일괄 갱신

**구현 함수**: `processNetworkCancel(db, paymentId, idempotencyKey)`
**Detector**: AtomicTransaction (`db.transaction(() => {...}); tx()` 패턴 — better-sqlite3 표준)

## 트리거 조건

- **When**: 외부 결제망 취소 이벤트 수신 (system trigger, 사용자 권한 무관)
- **If**: `payment.status === 'PAID'` AND `idempotencyKey` 미사용 (`payments.idempotency_key` 검색 결과 없음)
- **Then**:
  1. `cancel_transactions` INSERT (`cancel_type='NETWORK'`, `status='COMPLETED'`, `requester_type='SYSTEM'`)
  2. `payments.status` → `'NETWORK_CANCELED'` + `canceled_at = now`
  3. `vouchers.balance += payment.amount` (잔액 복구)
  4. 위 3단계 모두 atomic transaction 내 실행
- **Else**:
  - 이미 처리됨 (idempotencyKey 발견) → `{cancelId, status: 'ALREADY_PROCESSED'}` 반환 (idempotent response)
  - PAID 아님 → `CancelError('E409', 'Payment not cancelable via network cancel', 409)` throw

## 신뢰도

- **Detector 신뢰도**: 85% (AtomicTransaction — better-sqlite3 표준 패턴)
- **표준 패턴 검증**: `db.transaction(() => { ... }); tx();` — Sprint 262 F429에서 정립된 보편 패턴
