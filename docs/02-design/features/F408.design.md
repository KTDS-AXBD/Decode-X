# F408 TD-47 해소 — evaluator R2 경로 skill-packages 직접 파싱 — Design

> **Summary**: F356-B evaluator의 데이터 로딩 계층을 production R2 `skill-packages/{id}.skill.json` 단일 번들 직접 파싱으로 전환. 기존 `spec-containers/` 경로는 회귀 안전성을 위해 fallback으로 보존. Adapter 한 겹 추가로 6기준 채점 엔진은 무수정 유지.
>
> **Plan**: `docs/01-plan/features/F408.plan.md` (AIF-PLAN-041)

## 2. SkillPackage → SpecContent 매핑 테이블

| SpecContent 필드 | SkillPackage 출처 | 변환 규칙 | 결손 시 처리 |
|------------------|-------------------|----------|-------------|
| `rules: string[]` | `policies[]` | `[policiesToOriginalRules(policies)]` (1개 markdown table) | policies는 min(1) 강제 → 항상 존재 |
| `originalRules: string[]` | `policies[]` | 동일 (rules와 같은 markdown) | 동일 |
| `emptySlotRules: string[]` | (없음) | `[]` | 빈 배열 — rubric source_consistency가 "참고 맥락" 처리 |
| `runbooks: string[]` | (없음) | `[]` | 빈 배열 — rubric comment_doc_alignment가 "부재" 판정 → 자동 0~0.3 |
| `tests: string[]` | (없음) | `[]` | 빈 배열 — rubric exception_handling이 "부재" 판정 → 자동 0~0.4 |
| `contractYaml: string` | `technicalSpec.apis[]` | `technicalSpecToContractYaml()` — best-effort YAML | technicalSpec optional → 없으면 `""` |
| `provenanceYaml: string` | `provenance` | `provenanceToYaml()` — sources/orgId/extractedAt/pipeline 포맷 | 항상 존재 |
| `skillName: string` | `metadata.domain + "-" + (subdomain ?? "default")` | concatenation | metadata.domain 항상 존재 |

### 2.1 `policiesToOriginalRules()` 출력 형식

```markdown
| ID       | condition                                          | criteria                | outcome                      | exception |
|----------|----------------------------------------------------|-------------------------|------------------------------|-----------|
| BL-001   | 회원 잔액이 충전 한도(50만원)를 초과하면              | 한도 초과 금액 산정       | 충전 거부 + 잔액 알림         | —         |
| BL-002   | 1일 충전 횟수가 5회를 초과하면                       | 일별 충전 카운터 증가     | 추가 충전 차단 + 휴면 안내     | —         |
| ...      | ...                                                | ...                     | ...                          | ...       |
```

**ID 매핑 규칙** (rubric "ID 문자열 일치 필수" 호환):
- `policy.code` 형식: `POL-LPON-CHARGE-001`
- 추출 규칙: 마지막 `-` 이후 SEQ 부분(`001`)을 분리 → `BL-001` prefix 부착
- 충돌 시(다른 TYPE의 동일 SEQ): `BL-{TYPE}-{SEQ}` 형식 fallback (e.g., `BL-CHARGE-001`)

**exception 컬럼**:
- SkillPackage policies에 exception 필드 없음 → 일관되게 `—` (em dash) 표기
- rubric 0.9+ 요구사항 "exception 열 완결성 필수 (실내용 또는 — 명시)" 충족

### 2.2 `provenanceToYaml()` 출력 형식

```yaml
sources:
  - documentId: doc-lpon-charge-001
    pageRef: "p.12-15"
  - documentId: doc-lpon-charge-002
organizationId: lpon
extractedAt: "2026-03-15T10:23:00Z"
pipeline:
  stages:
    - svc-ingestion
    - svc-extraction
    - svc-policy
  models:
    extraction: anthropic/claude-sonnet-4-5
    policy: anthropic/claude-opus-4-5
businessRules:
  - id: BL-001
    source: doc-lpon-charge-001
  - id: BL-002
    source: doc-lpon-charge-001
```

**핵심**: `businessRules` 섹션을 명시적으로 포함하여 rubric source_consistency가 "provenance ↔ originalRules ID 1:1 매핑" 판정을 수행 가능하도록 함. policies의 source.documentId를 묶어서 자동 생성.

### 2.3 `technicalSpecToContractYaml()` (best-effort)

```yaml
# Auto-generated from skill-package.technicalSpec
contracts:
  - api: POST /lpon/charge
    given:
      memberId: string
      amount: number (KRW)
    when: 충전 요청
    then:
      success:
        balance: updated
        receiptId: uuid
      failure:
        code: CHARGE_LIMIT_EXCEEDED
```

- technicalSpec 부재 시: `# (no technicalSpec available — io_structure 평가는 rules의 outcome 필드 기반)` (rubric LLM에게 명시 컨텍스트 전달)

## 3. Code Diff (요약)

### 3.1 `services/svc-skill/src/ai-ready/spec-content-adapter.ts` (신규)

```ts
import { SkillPackageSchema, type SkillPackage, type Policy, type Provenance, type TechnicalSpec } from "@ai-foundry/types";
import type { SpecContent } from "./prompts.js";

export interface AdaptedSpec {
  specContent: SpecContent;
  skillName: string;
}

export function skillPackageToSpecContent(pkg: SkillPackage): AdaptedSpec {
  const skillName = `${pkg.metadata.domain}-${pkg.metadata.subdomain ?? "default"}`;
  const rulesMarkdown = policiesToOriginalRules(pkg.policies);
  return {
    skillName,
    specContent: {
      rules: [rulesMarkdown],
      originalRules: [rulesMarkdown],
      emptySlotRules: [],
      runbooks: [],
      tests: [],
      contractYaml: technicalSpecToContractYaml(pkg.technicalSpec),
      provenanceYaml: provenanceToYaml(pkg.provenance, pkg.policies),
    },
  };
}

export function policiesToOriginalRules(policies: Policy[]): string {
  const header = "| ID       | condition | criteria | outcome | exception |\n|----------|-----------|----------|---------|-----------|";
  const rows = policies.map((p) => {
    const seq = p.code.split("-").pop() ?? "000";
    const id = `BL-${seq}`;
    return `| ${id.padEnd(8)} | ${escapeMd(p.condition)} | ${escapeMd(p.criteria)} | ${escapeMd(p.outcome)} | — |`;
  });
  return [header, ...rows].join("\n");
}

export function provenanceToYaml(prov: Provenance, policies: Policy[]): string {
  const sourceLines = prov.sourceDocumentIds.map((id) => `  - documentId: ${id}`).join("\n");
  const businessRulesLines = policies
    .map((p) => `  - id: BL-${p.code.split("-").pop()}\n    source: ${p.source.documentId}`)
    .join("\n");
  const stageLines = prov.pipeline.stages.map((s) => `    - ${s}`).join("\n");
  const modelLines = Object.entries(prov.pipeline.models).map(([k, v]) => `    ${k}: ${v}`).join("\n");
  return [
    "sources:",
    sourceLines,
    `organizationId: ${prov.organizationId}`,
    `extractedAt: "${prov.extractedAt}"`,
    "pipeline:",
    "  stages:",
    stageLines,
    "  models:",
    modelLines,
    "businessRules:",
    businessRulesLines,
  ].join("\n");
}

export function technicalSpecToContractYaml(spec?: TechnicalSpec): string {
  if (!spec || spec.apis.length === 0) {
    return "# (no technicalSpec available — io_structure 평가는 rules의 outcome 필드 기반)";
  }
  const apis = spec.apis.slice(0, 5).map((a) => {
    return `  - api: ${a.method ?? "POST"} ${a.path ?? "/"}\n    given: {}\n    when: ${a.summary ?? "(unspecified)"}\n    then: {}`;
  }).join("\n");
  return `contracts:\n${apis}`;
}

function escapeMd(s: string): string {
  return s.replace(/\|/g, "\\|").replace(/\n/g, " ");
}
```

+   env: Env,

describe("skillPackageToSpecContent", () => {
  it("converts policies to BL-XXX markdown rules", () => {
    const pkg = makeFixturePackage({ policies: [policy("POL-LPON-CHARGE-001"), policy("POL-LPON-CHARGE-002")] });
    const { specContent } = skillPackageToSpecContent(pkg);

    expect(specContent.originalRules?.[0]).toContain("| — |");

  it("includes businessRules in provenanceYaml for source_consistency", () => {
    const pkg = makeFixturePackage();

    expect(specContent.provenanceYaml).toContain("businessRules:");
    expect(specContent.provenanceYaml).toContain("- id: BL-");
  });

  it("emits placeholder contractYaml when technicalSpec is absent", () => {
    const pkg = makeFixturePackage({ technicalSpec: undefined });
    const { specContent } = skillPackageToSpecContent(pkg);
    expect(specContent.contractYaml).toContain("(no technicalSpec available");
  });

  it("returns empty arrays for runbooks/tests/emptySlotRules", () => {
    const pkg = makeFixturePackage();
    const { specContent } = skillPackageToSpecContent(pkg);
    expect(specContent.runbooks).toEqual([]);
    expect(specContent.tests).toEqual([]);
    expect(specContent.emptySlotRules).toEqual([]);
  });

  it("derives skillName from metadata.domain + subdomain", () => {
    const pkg = makeFixturePackage({ metadata: { domain: "lpon", subdomain: "charge" } });

    expect(skillName).toBe("lpon-charge");
