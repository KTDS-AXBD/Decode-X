# Runbook — CS-005 연체 반납 일괄 배치

## 알림 조건
일별 batch 실행 결과 `overdueCount` 급증 (예: 평균 대비 3σ 초과) 시.

## 진단
1. 연체 패턴: 특정 풀 / 시간대 / 차종 군집 확인
2. 픽업 후 미반납 비율 (sessions.active → overdue 전환률)
3. 정산 시스템 장애 가능성 확인 (CS-006 batch 실패와의 연관성)

## 해소
- 임시: 연체 차량 추적 + 회원 연락 + 추가 요금 부과 (rental_payments 추가 INSERT)
- 영구: GPS 기반 자동 알림 + 회원권 등급 강등 정책 + 차량 회수 SLA 정립
