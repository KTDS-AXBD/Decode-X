# Runbook — CC-006: 결제 취소 한도 복구

## 트리거
가맹점 취소 또는 사용자 결제 취소 요청 (`POST /api/v1/credit-cards/transactions/:id/cancel`)

## 처리 흐름
1. `SELECT card_id, amount, status FROM card_transactions WHERE id=?`
2. `status != 'approved'` → `E409-TX`
3. **Atomic transaction**:
   - `UPDATE card_transactions SET status='cancelled', cancelled_at=now`
   - `UPDATE credit_cards SET used_amount = used_amount - amount`
4. `{transactionId, restoredAmount, cancelledAt}` 반환

## 트러블슈팅
- used_amount 음수 우려 → 사전 transaction.amount 양수 보장 + transaction 내부 read-then-update
- Race condition (CC-003 + CC-006 동시) → SQLite single-writer 보장

## 모니터링
- 일별 취소율 (cancelTransaction / approvePayment)
- 평균 restoredAmount
