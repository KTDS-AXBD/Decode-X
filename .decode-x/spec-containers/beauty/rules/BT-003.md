# BT-003 — 미용 예약 확정 Atomic 트랜잭션

**함수**: `confirmAppointment()`
**Detector**: AtomicTransaction

## 조건
미용 예약 확정 요청 시 스타일리스트 가용 여부 검사 후 atomic 처리.

## 기준
`stylists.status = 'available'` 확인 후 단일 트랜잭션으로:
1. appointments INSERT
2. stylists UPDATE (status='booked')
3. appointment_payments INSERT

## 결과
- 가용: atomic 예약 확정 (3개 테이블 동시 갱신)
- 불가: `E404-STYLIST` 예외

## 근거
예약 확정 시 스타일리스트 이중 배정 방지 — atomic 처리로 race condition 차단.
