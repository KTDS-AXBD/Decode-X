# SP-003 — 티켓 판매 Atomic 트랜잭션

**함수**: `processTicketSale()`
**Detector**: AtomicTransaction

## 조건
티켓 판매 요청 시 좌석 hold 상태 확인 후 원자 처리.

## 기준
`venue_seats.status = 'held'` — held 상태 좌석만 판매 가능.

## 결과
- atomic: ticket_sales INSERT + venue_seats UPDATE(sold) + issued_tickets INSERT + ticket_sales.status UPDATE(issued)
- 실패: `E404-SEAT` (held 좌석 미존재)

## 근거
이중 판매 방지 — 결제·좌석확정·발권을 하나의 트랜잭션으로 보장.
