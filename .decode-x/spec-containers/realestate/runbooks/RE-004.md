# Runbook — RE-004: 계약 상태 전환

## 트리거
- 만료 임박 → expiring (RE-005 batch 또는 manual)
- 양 당사자 합의 해지 → terminated
- 갱신 합의 → renewed

## 처리 흐름
1. 현재 status 조회
2. 허용 매트릭스 검증
3. `UPDATE leases SET status=?, terminated_at=?` (terminated 시점만)
4. `{previousStatus, newStatus}` 반환

## 알림
- expiring 마킹 시 양 당사자 SMS/email + 갱신 안내
- terminated 시 deposit 환불 트리거 (외부 결제)

## 모니터링
- 일별 상태 전환 카운트 per status
- 갱신률 (renewed / 총 expiring)
