# WL-006 — 취소 + 패널티 + 환불 Atomic 트랜잭션

**함수**: `processCancellationFee()`
**Detector**: AtomicTransaction

## 조건
예약 취소 요청 시 (appointments.status != 'cancelled').

## 기준
4개 DB 연산이 단일 트랜잭션으로 원자적 실행:
1. appointments.status → 'cancelled' UPDATE
2. cancellation_logs INSERT (패널티 + 환불 기록)
3. refund_records INSERT (환불 처리)
4. sessions.booked_count -1 UPDATE

## 결과
- 성공: 모든 연산 커밋 (자리 반환 + 환불 확정)
- 실패: 전체 롤백 (부분 취소 방지)

## 근거
취소 처리 시 자리 반환(booked_count 감소)과 환불이 동시에 성공해야 유효. 환불 실패 시 자리만 반환되는 불일치 방지.
