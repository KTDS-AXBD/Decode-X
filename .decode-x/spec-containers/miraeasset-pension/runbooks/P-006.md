# Runbook — P-006: 해지 처리

**Rule**: 진행 중 인출 없을 시 계좌 'TERMINATED' 전환 + 잔여 원리금 환급

## 트리거
- `POST /api/v1/pension/{accountId}/terminate`

## 전제조건
- `account.status`: 현재 계좌 상태 (`ACTIVE` | `SUSPENDED` | `RECEIVING`)
- 진행 중 중도인출 건 존재 여부 (`pension_withdrawals.status = 'UNDER_REVIEW'`)

## 처리 흐름
1. 진행 중 인출 건 조회 → `count > 0`이면 `E409-WITHDRAWAL_IN_PROGRESS` 반환
2. `account.status` 가 이미 `TERMINATED`이면 `E409-ALREADY_TERMINATED` 반환
3. 상태 전환: `ACTIVE/SUSPENDED/RECEIVING` → `TERMINATED`
4. 잔여 원리금 지급 트리거 (P-007 호출) + `PensionTerminated` 이벤트
