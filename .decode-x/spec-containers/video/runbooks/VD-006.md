# Runbook — VD-006 리워드 회수 atomic

## 알림 조건
`E404-BANNED-STREAM` 또는 회수 잔액 불일치 (`reward_clawback_records.status='clawed_back'`인데 `reward_clawbacks` row 누락) 의심 시.

## 진단
1. `video_streams.status='banned'` 조건 충족 확인 (정상 종료된 retired 스트림은 회수 대상 아님)
2. `clawbackAmount = rewardCost × clawbackRate` 계산 검증 (소수점 2자리 반올림)
3. `db.transaction()` rollback 로그 — atomic 실패 시 records/clawbacks 양쪽 모두 미생성

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: clawback rate 정책 (50% / 70% / 100%) 별 audit log 추가 + 회수 사유 enum 도입
