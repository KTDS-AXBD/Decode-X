# Runbook — RE-003: 계약 체결 atomic

## 트리거
양 당사자 계약 합의 → `POST /api/v1/realestate/leases/sign`

## 처리 흐름
1. property.status listed 검증
2. **Atomic transaction**:
   - `INSERT INTO leases (status='active', start_at, end_at)`
   - `UPDATE properties SET status='leased'`
3. `{leaseId, status, startAt, endAt}` 반환

## 트러블슈팅
- atomic rollback 시 property 'listed' 유지
- 중복 임차 위험 → atomic 필수

## 모니터링
- 일별 신규 임차 카운트
- 평균 lease 기간 (months)
