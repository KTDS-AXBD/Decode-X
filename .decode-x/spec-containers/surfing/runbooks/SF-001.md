# Runbook — SF-001 스팟 동시 active board 한도

## 알림 조건
`E422-SPOT-CAPACITY-EXCEEDED` 빈도 급증 시.

## 진단
1. `spots.active_boards` vs `total_capacity` 비교 조회
2. 특정 스팟/성수기(여름 주말/태풍 시즌 좋은 파도) 동시 입수 집중 여부 확인
3. `MAX_CONCURRENT_ACTIVE_BOARDS_PER_SPOT` 기본값(300) 적절성 검토 (해안선 길이/파도 break 수 기준)

## 해소
- 임시: 스팟에 finished 후보 board 정리 권유 또는 total_capacity 직접 조정
- 영구: 시간대 분산 입수권/예약제 도입 + 추가 인근 스팟 안내 + 안전요원 추가 배치
