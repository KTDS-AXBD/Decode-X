# Runbook — AQ-004 admit 상태 전환

## 알림 조건
`E409-SCHEDULE` 빈도 급증 시 (잘못된 상태 전환, 특히 긴급 closed).

## 진단
1. 운영 시스템 측 상태 캐시 stale 여부 확인 (앱에서 closed admit을 toured처럼 표시)
2. 동시 transition 충돌 — 두 운영자가 동시에 같은 schedule_id를 close/cancel 전환
3. 수족관 긴급 폐쇄/생물 점검 워크플로 매트릭스 검토 + 관람객 안내 정책 audit (수질/생물 건강 cross-check)

## 해소
- 임시: 사용자 안내 + 앱/안내판 refresh 강제 + closed notice 송출 (SMS + 푸시 + 매표소)
- 영구: optimistic locking + closed 시 인접 어트랙션 추천 + 관람객 보상 자동 스케줄 hook
