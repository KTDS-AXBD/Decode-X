# Runbook — SH-003 화물 적재 atomic 실패

## 알림 조건
`E404-BOOKING` 빈도 급증 또는 `cargo_loads`/`voyage_bookings`/`freight_payments` 정합성 깨짐.

## 진단
1. `voyage_bookings.status = 'confirmed'` 조건 만족 여부 (적재 직전 상태)
2. atomic transaction 실패 시 3 테이블 일치 여부 (`db.transaction` rollback)
3. 동일 `bookingId` 중복 적재 시도 여부 (idempotency)

## 해소
- 임시: 실패 booking은 transitionBookingStatus로 상태 회복 후 재시도
- 영구: confirmed 상태 ttl 정책 + 중복 적재 방지 unique index
