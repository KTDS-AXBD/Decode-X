# Runbook — SH-006 체선료 환불 atomic 실패

## 알림 조건
`E404-ARRIVED-CARGO` 빈도 급증 또는 `demurrage_refund_records`/`demurrage_refunds` 정합성 깨짐.

## 진단
1. `cargo_loads.status = 'arrived'` 조건 만족 여부 (환불 대상 정확성)
2. atomic transaction 실패 시 3 테이블 일치 여부 (`db.transaction` rollback)
3. 동일 `cargoLoadId` 중복 환불 시도 여부 (idempotency)

## 해소
- 임시: 실패 환불 케이스 화주에게 안내 + 수동 보정 절차 안내
- 영구: 중복 환불 방지 unique index + arrived 상태 환불 가능 기한 명시
