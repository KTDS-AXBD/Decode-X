# Runbook — PH-002: 처방 잔여 횟수 한도 비교

## 트리거
약사/처방SYSTEM → `GET /api/v1/pharmacy/prescriptions/:id/refill-quota`

## 처리 흐름
1. 처방의 약품 유형(category) 조회
2. 유형별 `refillQuotaLimit` 산출 (antibiotic:1 / painkiller:3 / chronic:12 / supplement:6)
3. `refillsUsed > refillQuotaLimit` 초과 시 차단

## 모니터링
- E422-REFILL-QUOTA 발생률 (처방 유형별 초과 비율)
- 만성질환 처방 리필 패턴 분석
