# Runbook — EX-003 부스 개장 atomic

## 알림 조건
`E404-SCHEDULE` 또는 atomic 부분 실패 (admissions INSERT 성공 + booth_payments INSERT 실패) 의심 시.

## 진단
1. `booth_schedules.status` 확인 — `booked` 상태에서만 개장 진입 가능
2. `db.transaction()` rollback 로그 확인
3. `admissions.schedule_id` ↔ `booth_payments.schedule_id` 정합성 cross-check + 부스 점검 인프라 모니터링

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: cross-table FK 강제 + 결제 PG fallback + 부스 점검 dual-staff 운영 설계
