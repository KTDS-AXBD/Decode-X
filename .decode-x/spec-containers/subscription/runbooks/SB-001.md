# Runbook — SB-001: 구독 생성

## 트리거
사용자 신규 결제 → `POST /api/v1/subscriptions/create`

## 처리 흐름
1. priceKrw 검증 (`≥ MIN_PRICE_KRW`)
2. cycleDays 검증 (`MIN_CYCLE_DAYS ≤ x ≤ MAX_CYCLE_DAYS`)
3. nextChargeAt 산정 (`now + cycleDays`)
4. `subscriptions` INSERT (status='active', cycle_count=0)

## 모니터링
- 가격대별 구독 분포 + 평균 cycleDays
- E422-* 발생률 (가격/주기 검증 실패)
