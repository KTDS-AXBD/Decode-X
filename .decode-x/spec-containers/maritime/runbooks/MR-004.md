# Runbook — MR-004: 선적 상태 전환

## 트리거
선사/선적관리SYSTEM → `PATCH /api/v1/maritime/shipments/:id/status`

## 처리 흐름
1. 현재 status 조회
2. 허용 매트릭스 검증 (booked→loaded, loaded→at_sea, at_sea→arrived, arrived→delivered)
3. status UPDATE + 타임스탬프 기록

## 상태 전환 타임스탬프
- loaded → loaded_at
- at_sea → departed_at
- arrived → arrived_at
- delivered → delivered_at

## 모니터링
- E409-SHIPMENT 발생률 (불법 상태 전환 시도)
- 각 상태별 체류 시간 (SLA 모니터링)
