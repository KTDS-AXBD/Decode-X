# WL-003 — 예약 확정 Atomic 트랜잭션

**함수**: `confirmAppointment()`
**Detector**: AtomicTransaction

## 조건
예약 확정 요청 시 (appointments.status = 'booked').

## 기준
4개 DB 연산이 단일 트랜잭션으로 원자적 실행:
1. appointments.status → 'confirmed' UPDATE
2. appointment_payments INSERT (결제 기록)
3. appointment_resources INSERT (자원 hold 기록)
4. resources.status → 'held' UPDATE

## 결과
- 성공: 모든 연산 커밋
- 실패: 전체 롤백 (부분 확정 방지)

## 근거
예약 확정은 결제 + 자원 예약이 동시에 성공해야 유효. 결제 후 자원 hold 실패 시 오버부킹 발생 가능.
