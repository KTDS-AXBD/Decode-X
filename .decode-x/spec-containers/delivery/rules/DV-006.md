# DV-006: 배송 취소 + 반품 atomic

**Rule**: SHIPPED/DELIVERED 상태 검증 후 atomic하게 status='RETURNED' + return_requests INSERT (refund=weight × 1,000원)

**구현 함수**: `cancelAndReturn`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();`)

## 트리거 조건
- **When**: 사용자 반품 요청 또는 배송 시스템 취소
- **If**: `delivery.status ∈ {SHIPPED, DELIVERED}`
- **Then**: atomic transaction
  1. `UPDATE deliveries SET status='RETURNED', cancelled_at=now`
  2. `INSERT INTO return_requests (refund_amount = floor(weight_kg × 1000))`
- **Else**: `E404-DV` 또는 `E409-DV` (PENDING/CANCELLED/RETURNED 등 불허 상태)

## 무결성 보장
- 두 변경 atomic — return_requests INSERT 실패 시 deliveries.status 변화 없음
- refund_amount 0 또는 음수 발생 우려 없음 (weight_kg ≥ MIN_WEIGHT_KG 보장)

## 신뢰도
- Detector: 85% (AtomicTransaction — better-sqlite3 표준)
