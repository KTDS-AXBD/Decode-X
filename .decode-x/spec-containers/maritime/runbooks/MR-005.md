# Runbook — MR-005: 항구 처리 일괄 갱신

## 트리거
항구관리SYSTEM → `POST /api/v1/maritime/port-handling/batch` (일괄 배치)

## 처리 흐름
1. status='arrived' AND arrived_at <= arrivalCutoff AND port_handled=0 조건 조회
2. 각 선적건에 대해:
   a. port_handled=1 갱신
   b. port_handled_at 기록
   c. status='port_cleared' 갱신

## 모니터링
- handledCount 일별 추이
- 항구 처리 대기 시간 (arrived_at → port_handled_at)
- 항구별 처리 처리량
