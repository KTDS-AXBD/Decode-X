# Runbook — NW-004 article 상태 전환

## 알림 조건
`E409-PUBLISH` 빈도 급증 시 (잘못된 상태 전환 시도, 특히 정정 보도 retracted 전환).

## 진단
1. 클라이언트 측 article 상태 캐시 stale 여부 확인 (검색 엔진 색인이 retracted article을 published처럼 표시)
2. 동시 transition 충돌 — 두 에디터가 동시에 같은 publish_id를 archive/retract 전환
3. 편집 워크플로 매트릭스 (drafted → edited 자동 vs 수동) 검토 + 정정 보도 정책 audit

## 해소
- 임시: 사용자 안내 + 색인 update 강제 + retracted notice 페이지 표시
- 영구: optimistic locking (version column) 도입 + retracted 시 SEO meta tag 자동 갱신 + Google News API 통보 hook
