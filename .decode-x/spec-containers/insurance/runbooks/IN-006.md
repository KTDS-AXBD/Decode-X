# Runbook — IN-006: 청구 거절 + 환불 atomic

## 트리거
심사관 거절 결정 (`POST /api/v1/insurance/claims/:id/reject-with-refund`)

## 처리 흐름
1. claim 조회 (policy_id, status, filed_at)
2. status pending 검증
3. policy 조회 (premium_krw)
4. elapsedDays 계산 (filed_at vs now)
5. refundAmount 산정 (`elapsedDays ≤ 30 ? premium : 0`)
6. **Atomic transaction**:
   - `UPDATE claims SET status='rejected', rejected_at=now`
   - (조건부) `INSERT INTO claim_payments (payment_type='refund', amount=premium)`
7. `{refundPaymentId, refundAmount, rejectedAt}` 반환

## 트러블슈팅
- atomic rollback 시 status 변화 없음 + claim_payments 흔적 없음
- refundPaymentId NULL인 경우 INSERT skip (status update만 commit)

## 모니터링
- 일별 거절률 (rejected / total filed)
- 평균 elapsedDays + 30일 이내 환불 비율
- 누적 환불 비용 (회계 입력)
