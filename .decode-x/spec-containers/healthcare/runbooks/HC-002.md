# Runbook — HC-002: 처방 dosage 한도 검증

## 트리거
의사 처방 직전 (의료 시스템에서 자동 호출)

## 처리 흐름
1. patient.status active 검증
2. 현재 active 처방의 daily_dosage 합산
3. dosageLimit = `MAX_DAILY_DOSAGE_MG - activeDosage`
4. requestedDosageMg 비교 (Path B var-vs-var with `limit`)
5. `{canPrescribe: true, remainingDosageHeadroom}` 반환

## 모니터링
- E422-LIMIT 발생률 (overdose 위험 사례 추적)
- drug_code별 평균 dosage 분포
- 의사별 처방 패턴 (신뢰도 입력)
