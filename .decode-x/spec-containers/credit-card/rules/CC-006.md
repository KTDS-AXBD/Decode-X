# CC-006: 결제 취소 → 한도 복구 (atomic)

**Rule**: 승인 transaction 취소 시 atomic하게 transaction status='cancelled' + used_amount 복구

**구현 함수**: `cancelTransaction`
**Detector**: AtomicTransaction (`db.transaction(() => { ... }); tx();`)

## 트리거 조건
- **When**: 가맹점 취소 또는 사용자 결제 취소 요청
- **If**: `transaction.status = 'approved'`
- **Then**: atomic transaction
  1. `UPDATE card_transactions SET status='cancelled', cancelled_at=now`
  2. `UPDATE credit_cards SET used_amount = used_amount - amount`
- **Else**: `E404-TX` (미존재) / `E409-TX` (이미 cancelled/refunded)

## 무결성 보장
- 두 UPDATE 모두 atomic → 실패 시 rollback
- used_amount 음수 발생 시 invariant 위반 (사전 transaction.amount 검증으로 회피)

## 신뢰도
- Detector: 85% (AtomicTransaction — better-sqlite3 표준)
