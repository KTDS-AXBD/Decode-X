# Runbook — ED-003: 수강 신청 atomic

## 트리거
학생 수강 신청 (`POST /api/v1/education/enrollments`)

## 처리 흐름
1. course 조회 (capacity, enrolled_count, status)
2. status open + capacity 여유 검증
3. **Atomic transaction**:
   - `INSERT INTO enrollments (status='enrolled')`
   - `UPDATE courses SET enrolled_count = enrolled_count + 1`
4. `{enrollmentId, status, enrolledAt}` 반환

## 트러블슈팅
- atomic rollback 시 capacity 누적 누락 방지
- over-enrollment 위험 → atomic 필수

## 모니터링
- 일별 수강 신청 카운트
- 과목별 enrolled_count vs capacity
