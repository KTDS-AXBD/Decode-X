# Runbook — SK-001 리조트 동시 active pass 한도

## 알림 조건
`E422-RESORT-CAPACITY-EXCEEDED` 빈도 급증 시.

## 진단
1. `resorts.active_passes` vs `total_capacity` 비교 조회
2. 특정 리조트/성수기(주말/연말연시/방학) 동시 입장 집중 여부 확인
3. `MAX_CONCURRENT_ACTIVE_PASSES_PER_RESORT` 기본값(8000) 적절성 검토 (안전 수용 인원/리프트 처리량 기준)

## 해소
- 임시: 리조트에 completed 후보 pass 정리 권유 또는 total_capacity 직접 조정
- 영구: 야간 운영 확대/시간대 분산 입장 + 추가 리프트 증설 + 슬로프 안전 수용 인원 재산정
