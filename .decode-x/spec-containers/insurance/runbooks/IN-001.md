# Runbook — IN-001: 보험 가입

## 트리거
사용자 신규 가입 신청 → `POST /api/v1/insurance/policies/issue`

## 처리 흐름
1. premium 검증 (`≥ MIN_PREMIUM_KRW`)
2. age 검증 (`MIN_INSURED_AGE ≤ age ≤ MAX_INSURED_AGE`)
3. expires_at 산정 (`now + 365일`)
4. `policies` INSERT (status='active', reserved_amount=0)

## 모니터링
- E422-* 발생률 (premium/age 검증 실패)
- 정책 type별 가입 분포 (LIFE/HEALTH/AUTO/PROPERTY)
