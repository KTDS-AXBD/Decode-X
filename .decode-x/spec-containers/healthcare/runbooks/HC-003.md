# Runbook — HC-003: 예약 잡기 atomic

## 트리거
사용자 예약 요청 (`POST /api/v1/healthcare/appointments/book`)

## 처리 흐름
1. slot 조회 (status, scheduled_at)
2. status available 검증
3. **Atomic transaction**:
   - `INSERT INTO appointments (status='booked')`
   - `UPDATE slots SET status='reserved'`
4. `{appointmentId, scheduledAt}` 반환

## 트러블슈팅
- atomic rollback 시 slot 잠금 안 됨
- slot 중복 reserved 위험 — atomic 필수

## 모니터링
- 일별 예약 카운트 + 시간대별 분포
- 의사별 예약 점유율 (가용성 분석)
