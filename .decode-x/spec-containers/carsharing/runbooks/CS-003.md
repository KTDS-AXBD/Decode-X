# Runbook — CS-003 차량 픽업 atomic

## 알림 조건
`E404-RESERVATION` (confirmed 예약 없음) 빈도 급증 또는 atomic transaction rollback 발생 시.

## 진단
1. `vehicle_reservations` 상태가 confirmed인 예약 존재 여부
2. atomic transaction 단계 실패 (rental_sessions INSERT / vehicle_reservations UPDATE / rental_payments INSERT)
3. 동시성 충돌 (같은 reservation 두 번 픽업 처리) 추적

## 해소
- 임시: 실패 reservation 수동 status 정합화 + rental_payments 누락 시 보정
- 영구: 픽업 인증(NFC/QR) 동시 호출 방지 + idempotency 키 도입
