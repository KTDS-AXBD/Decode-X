# Runbook — DV-004: 배송 상태 전환

## 트리거
- 정상 완료 (택배사 수령 확인) → DELIVERED
- 사용자 사전 취소 (PENDING) → CANCELLED
- 배송 중 취소 (SHIPPED) → CANCELLED
- 반품 (SHIPPED) → RETURNED

## 처리 흐름
1. 현재 status 조회
2. 허용 매트릭스 검증
3. `UPDATE deliveries SET status=?, <field>=?`
4. `{previousStatus, newStatus}` 반환

## 알림
- DELIVERED 시 사용자 SMS/email 즉시 발송
- CANCELLED 시 환불 트리거 (외부 결제 시스템)

## 모니터링
- 일별 상태 전환 카운트 per status
- 평균 배송 lead time (DELIVERED.delivered_at - SHIPPED.shipped_at)
