# Runbook — SF-003 서핑 세션 시작 atomic

## 알림 조건
`E404-SCHEDULE` 또는 atomic 부분 실패 (boards INSERT 성공 + board_payments INSERT 실패) 의심 시.

## 진단
1. `session_schedules.status` 확인 — `reserved` 상태에서만 세션 진입 가능
2. `db.transaction()` rollback 로그 확인
3. `boards.schedule_id` ↔ `board_payments.schedule_id` 정합성 cross-check + 입수 게이트 RFID reader 모니터링

## 해소
- 임시: 부분 실패 row 수동 정리 후 재시도
- 영구: cross-table FK 강제 + 입수 reader fallback (백업 단말) + 모바일 pass dual-channel 설계
