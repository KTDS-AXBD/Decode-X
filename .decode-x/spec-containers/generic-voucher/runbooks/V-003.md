# Runbook — V-003: 사용 시 잔액 차감

## 트리거
- `POST /api/v1/vouchers/{id}/redeem` 호출

## 처리 흐름 (atomic)
1. voucher row 조회 + balance 검증
2. db.transaction 시작
3. UPDATE balance (차감) + UPDATE used_count
4. INSERT voucher_ledger_entries (DEBIT_REDEEM)
5. 무결성 검증 (after row exists?) → 실패 시 throw → rollback

## 모니터링
- ledger / vouchers 합계 일치 검증 (정합성 batch)
- transaction failure rate
