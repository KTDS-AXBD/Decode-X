# Runbook — IN-004: 정책 상태 전환

## 트리거
- 보험료 미납 → suspended (관리 시스템)
- 미납 해소 → active (재개)
- 사용자 해약 → cancelled

## 처리 흐름
1. 현재 status 조회
2. 허용 매트릭스 검증
3. `UPDATE policies SET status=?, cancelled_at=?` (cancelled 시점만)
4. `{previousStatus, newStatus}` 반환

## 알림
- suspended 전환 시 사용자 SMS/email + 보험료 납부 안내
- cancelled 전환 시 환불 정책 안내

## 모니터링
- 일별 상태 전환 카운트 per status
- 미납률 (suspended / active)
