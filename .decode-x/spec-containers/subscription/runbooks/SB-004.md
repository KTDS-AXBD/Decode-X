# Runbook — SB-004: 구독 상태 전환

## 트리거
- 사용자 일시 중지 (active → paused)
- 사용자 재개 (paused → active)
- 사용자 취소 (active/paused → cancelled, 환불 없음 시 SB-004로 단독)

## 처리 흐름
1. 현재 status 조회
2. 허용 매트릭스 검증
3. `UPDATE subscriptions SET status=?, cancelled_at=?` (cancelled 시점만 cancelled_at 설정)
4. `{previousStatus, newStatus}` 반환

## 알림
- paused 전환 시 사용자 SMS/email 통지
- cancelled 전환 시 환불 정책 안내 (SB-006로 환불 별도 트리거 가능)

## 모니터링
- 일별 상태 전환 카운트 per status
- pause → cancel 전환률 (이탈 분석)
