# SP-006 — 환불 + 재예약 Atomic 트랜잭션

**함수**: `processRefundRebook()`
**Detector**: AtomicTransaction

## 조건
환불 + 재예약 요청 시 원본 판매 존재 확인 후 원자 처리.

## 기준
원본 판매 ticket_sales 존재 필수.

## 결과
- atomic: refund_records INSERT + ticket_sales UPDATE(cancelled) + venue_seats UPDATE(available) + ticket_sales INSERT(rebook) + cancellation_logs INSERT + venue_seats UPDATE(sold)
- 실패: SportsError

## 근거
환불 일관성 — 취소·환불·좌석반납·재예약을 하나의 트랜잭션으로 보장.
