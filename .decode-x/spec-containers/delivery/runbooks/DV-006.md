# Runbook — DV-006: 배송 취소 + 반품 atomic

## 트리거
사용자 반품 요청 (`POST /api/v1/deliveries/:id/return`)

## 처리 흐름
1. delivery 조회 (order_id, weight_kg, status)
2. status SHIPPED/DELIVERED 검증
3. refund_amount 계산 (`weight_kg × 1,000`)
4. **Atomic transaction**:
   - `UPDATE deliveries SET status='RETURNED', cancelled_at=now`
   - `INSERT INTO return_requests (refund_amount, ...)`
5. `{returnRequestId, refundAmount, cancelledAt}` 반환

## 트러블슈팅
- atomic rollback 시 deliveries.status 변화 없음 + return_requests 흔적 없음
- refund_amount 0 우려 없음 (weight ≥ MIN_WEIGHT_KG 보장)

## 모니터링
- 일별 반품률 (cancelAndReturn / 정상 DELIVERED)
- 평균 refundAmount + 누적 refund 비용
