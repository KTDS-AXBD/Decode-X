# Runbook — PH-003: 처방 조제 atomic

## 트리거
약사/조제SYSTEM → `POST /api/v1/pharmacy/prescriptions/:id/dispense`

## 처리 흐름
1. 처방 status='pending' 확인
2. 약품 status != 'recalled' 확인
3. stock_count > 0 확인
4. atomic: stock_count 차감 + prescriptions.status='dispensed' + dispenses INSERT

## 모니터링
- E422-OUT-OF-STOCK 발생률 (재고 소진 약품 비율)
- E422-DRUG-RECALLED 발생률 (리콜 약품 조제 시도)
- 조제 트랜잭션 성공률
